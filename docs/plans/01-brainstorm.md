# Brainstorm — 학생 용돈 기입장

> 원격 무인 세션 — AI 자체 판단으로 결정. 사용자 질의 없음.

## MUST (꼭 필요한 기능)
1. **거래 입력 폼** — 날짜·종류(수입/지출)·항목명·금액·카테고리. 키보드 Enter 제출, 빈 값/음수 검증.
2. **거래 리스트** — 최신순. 각 줄에 삭제 버튼. 카테고리 색 칩 표시.
3. **이번 달 합계 카드 3개** — 수입 / 지출 / 잔액 (잔액 음수면 경고색).
4. **카테고리 도넛 차트** — 지출 분포. 호버 시 비율 툴팁.
5. **주차 바 차트** — 이번 달 1~5주 차의 지출 합계.
6. **localStorage 자동 저장 + 복원** — 새로고침해도 유지.
7. **JSON 내보내기** — 텍스트 파일 다운로드. 학교 PC 간 백업용.
8. **월 선택 셀렉터** — 현재 월 외 과거 월도 조회 가능 (드롭다운).

## SHOULD (있으면 좋은 기능)
1. **JSON 가져오기** — 백업 파일 업로드 시 병합.
2. **카테고리 프리셋** — 학용품 / 간식 / 교통 / 저축 / 선물 / 기타 (이모지 포함).
3. **빈 상태 가이드** — 첫 사용자에게 안내 메시지.
4. **요약 텍스트** — "이번 달 지출 1위 카테고리는 간식입니다" 자동 문구.

## MUST NOT (절대 금지)
1. **실결제·은행 API 연동** — scope creep + 미성년 보호.
2. **가족·친구와 공유 기능** — 사회·심리 문제 잠재.
3. **클라우드 동기화** — 개인정보 외부 전송 금지.
4. **로그인·회원가입** — 학생 식별 정보 수집 금지.
5. **외부 광고·트래커** — 학생 대상 앱에 부적절.
6. **금융 권유성 문구** — "투자", "이자 받기" 등.

## scope 제한 정당화
- 학생용 금융 교육은 "내가 직접 본다"가 핵심. 공유/동기화는 다음 단계 토픽.
- 1:1 환경 spec — 멀티 유저 가정 불필요.

## 사용자 흐름 (golden path)
1. 첫 진입 → 빈 상태 카드 ("첫 거래를 입력해 보세요") + 입력 폼 포커스.
2. 거래 1건 입력 → Enter / 추가 버튼 → 리스트·차트·합계 즉시 갱신.
3. 5건 이상 → 도넛/바 차트가 의미 있는 분포 표시.
4. JSON 내보내기 → `용돈기입장-YYYY-MM.json` 다운로드.
